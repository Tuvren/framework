# Stage 2 (Architecture) Changelog

Local SemVer history for `.constitution/architecture/`.

- v0.10.1 - Reviewed for upstream delta; parallel structural migration tracked in GH issue #87 (Tuvren/runtime, "Epic — Restructure into a Sovereign Polyglot Framework"). That epic is explicitly not a `.constitution/` epic (status and milestone tracking live on the GitHub issue itself, per its own §12) and is out of scope for the four-stage planning framework; it relocates and renames physical implementation trees without changing any logical container boundary, communication style, or trust boundary described here. No layer-specific changes required. M10 of that epic is the point at which `.constitution/` is rewritten to reflect the post-migration repository; any files in this directory citing `boundaries/*` physical paths are stale-pending-M10-rewrite in the interim.
- v0.10.0 - Realized the SaaS-Readiness program (PRD v0.10.0, CAP-P0-064 through CAP-P0-070) in the logical layer without adding or removing any container. Added five core architectural principles to `strategy.md` (tenancy-agnostic scope seam; isolation-by-construction with a scope-free kernel syscall surface; reclaimable + erasable durable state; runtime-owned conversation state; backend-clocked execution authority) and five failure classes (cross-scope leak, unbounded-growth/un-erasable-history, lease-clock-skew split-brain, provider-state coupling, freeze-before-scope). Extended the **Durable State Boundary** with a construction-bound Scope, isolation-by-construction, and capability-descriptor bits for shared-lease-clock and reclamation; extended the **Kernel Boundary** with a reachability-based reclamation primitive (mechanism, grace-windowed) while keeping the syscall surface and gRPC interop scope-free (resolved fork Q1: scope is a substrate binding, not a syscall argument); extended the **Provider Gateway** with conversation-state ownership (provider state = reconstructable optimization); extended the **Client Endpoint Boundary** with the client-result-as-proposal invariant and the idempotency envelope / two-lease composition; and added matching §2.2 boundary notes. Added six resilience models to `resilience.md` (Tenancy & Scope Isolation; Data Lifecycle, Reclamation & Erasure; Backend-Authoritative Lease Clock; Side-Effect-Once Under Preemption; Conversation-State Ownership; SDK Stability & Distribution Sequencing — freeze-after-tenancy+GC, resolved fork Q3) and extended the Secret Isolation Model to cover shreddable untrusted-edge payloads with host-held keys. Added five logical risks to `risks.md` (cross-scope leak, unbounded-growth/erasure, lease-clock-skew, provider-state coupling, freeze-before-scope) and two critical flows (`flow-tenant-offboarding-erasure.md` §4.17 for CAP-P0-066/067, `flow-preemption-idempotent-side-effect.md` §4.18 for CAP-P0-068). Stayed strictly logical: no concrete vendors, schemas, or package identifiers (those bind in TechSpec). Traces to PRD CAP-P0-064 through CAP-P0-070.
- v0.9.1 - Migrated the monolithic `constitution/Architecture.md` into the modular `.constitution/architecture/` directory (strategy, containers, resilience, risks, and 16 flows/). Content preserved verbatim; container heading slugs preserved for authority-packet anchors (e.g. `mcp-client-container`).
- v0.9.0 - Realized the capability-orchestration model in the logical layer: reframed the single Tool Execution Gateway into a capability-orchestration surface by adding the Capability Registry, Binding & Endpoint Resolver, and Capability Policy Engine containers plus the Client Endpoint Boundary for the Tuvren-client execution class; described the four execution classes (provider-native, provider-mediated developer-provided, Tuvren-server, Tuvren-client) as ownership/observation profiles; classified MCP as a binding mechanism across classes; and added the compositional-model principle, execution-class trust relationships and failure classes, three critical flows (exposure-time tool-surface planning, invocation-time binding/policy with provider-native attribution, Tuvren-client lease/dispatch), and matching resilience models and logical risks. Traces to PRD CAP-P0-056 through CAP-P1-063. Existing tool execution becomes the Tuvren-server endpoint; no existing container is removed.
- v0.8.0 - Realized the production-trust block in the logical layer: added the Telemetry & Observability Boundary container (correlated operational-telemetry surface plus vendor-neutral export edge, distinct from the real-time host event stream) tracing to CAP-P0-052 / CAP-P1-053; added framework-owned Execution Bound enforcement (CAP-P0-054) and a cross-cutting Secret Isolation Model (CAP-P0-055); added a Recovery & Durability Verification Model with a verification-time fault-injection seam and crash-recovery conformance backing the sharpened Reliability guarantee; added matching architectural principles, trust relationships, failure classes, three critical flows (crash-recovery, telemetry capture/export, bounded-execution stop), and logical risks.
- v0.7.1 - Clarified current Reference Host posture after Epics AM-AT landed: durable reads now remove the kernel-inspector seam, the MCP client is first-class, and the retired playground host has been consolidated into a REPL CLI with headless mode, streaming JSONL output, and transcript replay.
- v0.7.0 - Added the Durable-Read Surface responsibility on Framework Shared Services, thread enumeration as a kernel structural primitive with backend-advertised capability, the Curated Host-Facing SDK container with shared-primitive plus slim-convenience split, the Schema Authoring Helper and MCP Tool Source responsibilities under the Tool Execution Gateway, and the headless plus transcript responsibilities on the Reference Host; promoted the SDK-only proving-host invariant from a risk-mitigation aspiration to a satisfied invariant; added new logical risks for the kernel-spec amendment cascade, schema-adapter detection ambiguity, MCP transport fragmentation, and durable-read pagination shape divergence.
